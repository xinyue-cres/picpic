End-to-end data flow from filesystem to UI to trash. Photos on disk are the immutable source. Scan reads paths only. Analyze reads paths from SQLite, decodes image bytes from disk, and writes feature columns back. Rules reads features and writes verdict. Web UI reads photos rows and streams file bytes to the browser; when the user trashes a photo, the Trash module moves the original into .picpic-trash/ and updates status='trashed'. The dashed edge from HuggingFace is the sole network hop, first-run only, guarded by the [clip] extra.
